ServiceNow Incident Automation with ML + API Workflow

This project is a Python automation system that reads incident records from ServiceNow, processes event-driven updates, predicts urgency with a machine learning model, and outputs dashboards and presentations for faster decision-making.

What The Code Does

End-to-end incident intelligence pipeline

The codebase simulates and manages an IT operations flow: events enter through an inbox/queue workflow, incidents are retrieved and interpreted from ServiceNow, urgency is scored via a trained model, and outputs are generated as both HTML dashboards and executive-ready slides.

  • Fetches incident data from ServiceNow table API endpoints.
  • Normalizes and routes incident events through file-based triggers.
  • Applies a custom urgency model for decision support.
  • Creates operational outputs that are useful for analysts and leadership.

How This Demonstrates ML

Predictive prioritization, not just reporting

The project includes model training and inference workflows that classify likely urgency levels. This demonstrates practical ML usage: converting incident attributes into predictive signals that help prioritize response before manual triage is complete.

How This Demonstrates AI

Decision support and intelligent automation

AI value appears in automated scoring and recommendation-style logic that augments analyst decision-making. Instead of manually sorting all records, analysts can focus on likely high-impact incidents first, reducing cognitive load and response time.

How This Demonstrates ServiceNow + API Skills

Enterprise integration pattern

The implementation uses authenticated REST calls to ServiceNow table APIs, then structures retrieved data for processing and visualization. This reflects a common enterprise pattern: external system integration, transformation, and orchestration inside a custom automation layer.

Example Use Case

What this is good for

A managed services team receives hundreds of incidents per week from multiple business units. Manual triage creates delays and inconsistent prioritization. With this project, the team can automatically pull incidents from ServiceNow, score urgency, and publish a daily dashboard + briefing deck. Result: faster escalation for high-risk issues, clearer operational visibility, and better leadership reporting.
